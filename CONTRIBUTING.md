# Contributing to Beloved

We welcome contributions of new component modules, assembly compiler plugins, CLI commands, and bug fixes!

## Code of Conduct

Please be respectful and constructive in all interactions.

## How to Contribute

1. **Fork the Repository**: Clone your fork locally.
2. **Create a Feature Branch**: Use descriptive names (e.g., `feature/analytics-plugin`).
3. **Write Tests**: Ensure any engine changes are fully verified and unit test coverage is maintained.
4. **Submit a PR**: Describe your changes clearly and target the `master` branch.

## Development Setup

Refer to the [README.md](./README.md) for details on running the local registries, database provisioning, and compiling components.
